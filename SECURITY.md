# Security Policy

## Supported Versions

We actively provide security updates for the following versions of this project:

| Version | Supported          |
| ------- | ------------------ |
| main    | ✅                |
| < 1.0   | ❌                |

## Reporting a Vulnerability

We take the security of our data and tools seriously. If you discover a security vulnerability, please do **not** open a public issue. Instead, follow the steps below:

1. **Private Disclosure**: Please report the vulnerability by opening a [Draft Security Advisory](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) on GitHub or by contacting the maintainers directly via the email listed in the repository profile.
2. **Details to Include**:
   - A description of the vulnerability.
   - Steps to reproduce the issue.
   - Potential impact of the vulnerability.
   - Any suggested fixes or mitigations.

## Our Response Process

- **Acknowledgment**: We will acknowledge receipt of your report within 48 hours.
- **Investigation**: We will investigate the issue and keep you updated on our progress.
- **Resolution**: Once a fix is developed, we will release a new version and, if appropriate, provide credit to the reporter.

## Data Privacy

If the vulnerability involves exposure of sensitive biological or clinical data, please prioritize immediate notification so we can take the necessary steps to protect data integrity and privacy. Please contact security-cn@patsnap.com.

Thank you for helping keep our community and data secure.
